## 섹션 1: useState — 블록 위치 기억

레이아웃: 좌(시각) - 우(설명) 2컬럼

┌──────────────────┬──────────────────────────┐
│  Canvas          │                          │
│                  │  useState                │  font-size: 28px, #00ff88
│  블록이 왼쪽 →   │                          │
│  오른쪽으로      │  "값이 바뀌면             │  font-size: 18px, 흰색
│  이동하는 애니메이션│  화면이 자동으로 다시    │
│                  │   그려집니다"             │
│  블록 아래에     │                          │
│  현재 좌표 표시   │  ─────────────────────  │
│  x: 240px        │                          │
│  y: 180px        │  일반 변수라면:           │  font-size: 15px, #aaa
│  angle: 45°      │  x = 240                │
│                  │  → 화면 변화 없음 ❌      │
│                  │                          │
│                  │  useState라면:           │
│                  │  const [x, setX] =       │
│                  │  useState(240)           │
│                  │  setX(300)               │
│                  │  → 화면 자동 업데이트 ✅  │
└──────────────────┴──────────────────────────┘

Canvas 애니메이션:
- L 블록이 왼쪽에서 오른쪽으로 이동
- 블록 아래에 실시간 x, y 좌표값 텍스트 표시
- 이동하면 좌표값도 함께 바뀌는 것을 보여줌

## 섹션 2: useEffect — 블록이 저절로 떨어짐

레이아웃: 좌(설명) - 우(시각) 2컬럼

┌──────────────────────────┬──────────────────┐
│                          │  Canvas          │
│  useEffect               │                  │
│                          │  블록이 천천히    │
│  "컴포넌트가 화면에        │  아래로 낙하      │
│   나타났을 때 딱 한 번     │                  │
│   자동으로 실행됩니다"     │  [▶ 시작]        │
│                          │  버튼 클릭 시     │
│  ─────────────────────── │  낙하 시작        │
│                          │                  │
│  쉽게 말하면:             │  [■ 정지]        │
│  "게임 시작 버튼을 누르면  │  버튼 클릭 시    │
│   블록이 떨어지기          │  낙하 멈춤       │
│   시작합니다"             │                  │
│                          │  → cleanup이      │
│  cleanup이란:            │  낙하를 멈추는    │
│  "게임이 끝나면            │  역할을 함        │
│   블록 낙하도 멈춥니다"    │                  │
└──────────────────────────┴──────────────────┘

Canvas 애니메이션:
- I 블록이 위에서 아래로 낙하
- [▶ 시작] 버튼: requestAnimationFrame 시작
- [■ 정지] 버튼: cancelAnimationFrame
- "시작 = mount, 정지 = cleanup" 시각화

용어 정리:
- "게임 루프" → "블록이 매 순간 조금씩 아래로 내려오는 반복 작업"
  (requestAnimationFrame = 1초에 60번 실행되는 타이머)
- "mount" → "컴포넌트가 화면에 나타남"
- "cleanup" → "컴포넌트가 화면에서 사라질 때 정리 작업"

## 섹션 3: useMemo — 불필요한 계산 줄이기

레이아웃: 좌(시각) - 우(설명) 2컬럼

┌──────────────────┬──────────────────────────┐
│  Canvas          │                          │
│                  │  useMemo                 │
│  블록이 움직이고  │                          │
│  충돌 계산이      │  "결과가 바뀔 때만        │
│  얼마나 자주      │   다시 계산합니다"        │
│  발생하는지       │                          │
│  카운터로 표시    │  ─────────────────────  │
│                  │                          │
│  useMemo 없음:   │  쉽게 말하면:            │
│  충돌 계산 847회 │  "블록이 움직일 때마다    │
│  (매 프레임)      │   충돌 계산을 처음부터   │
│                  │   다시 하면 느려집니다.   │
│  useMemo 있음:   │   바뀐 것만 다시 계산    │
│  충돌 계산 43회   │   합니다"               │
│  (블록 이동 시만) │                          │
└──────────────────┴──────────────────────────┘

Canvas 애니메이션:
- 여러 블록이 쌓여있고 새 블록이 낙하
- 충돌 계산이 일어날 때 빨간 점 표시
- useMemo 없음 토글: 매 프레임 모든 블록에 빨간 점 (많음)
- useMemo 있음 토글: 움직이는 블록 근처만 빨간 점 (적음)
- [useMemo OFF] [useMemo ON] 토글 버튼

## 구현 방법

모든 Canvas는 별도 HTMLCanvasElement로 각 섹션에 삽입.
각 섹션 Canvas에 자체 requestAnimationFrame 루프.
섹션이 뷰포트에 들어올 때 애니메이션 시작,
벗어날 때 cancelAnimationFrame (IntersectionObserver 사용).

섹션 간 전환:
- 세로 스크롤 (overflow-y: scroll)
- 각 섹션 높이: 100vh
- 스크롤 snap: scroll-snap-type: y mandatory
- 섹션마다: scroll-snap-align: start
