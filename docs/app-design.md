## 전체 구조

```
index.html (SPA 진입점)
│
├── 네비게이션: [🎮 Play] [🧩 Why Tetris] [📊 Flamegraph] [⚙️ Lifecycle]
│
├── Tab 1: Play           ← 물리 테트리스 게임 플레이
├── Tab 2: Why Tetris     ← 주제 선정 이유 + 컴포넌트 시각화
├── Tab 3: Flamegraph     ← 렌더링 메트릭 패널
└── Tab 4: Lifecycle      ← React 생명주기 시각화
```

## Tab 1 — 🎮 Play (테트리스 게임)

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  [ 🎮 Play ] [ 🧩 Why ] [ 📊 Flame ] [ ⚙️ LC ] │
├──────────────────┬──────────────────────────┤
│                  │  HOLD (R키)              │
│   TETRIS BOARD   │  [ 보관 블록 표시 ]      │
│   (물리 기반)    │  (없으면 빈 칸)          │
│                  │                          │
│                  │  NEXT                    │
│                  │  [ 다음 블록 미리보기 ]  │
│                  │                          │
│                  │  SCORE: 0                │
│                  │  LEVEL: 1                │
│                  │  LINES: 0                │
│                  │                          │
│                  │  [▶ START] [⏸ PAUSE]    │
│                  │                          │
│                  │  ← →  : 좌우 이동        │
│                  │  ↑     : 90도 즉시 회전  │
│                  │  Q     : 반시계 자유 회전 │
│                  │  E     : 시계 자유 회전   │
│                  │  ↓     : 소프트 드롭     │
│                  │  SPC   : 하드 드롭       │
│                  │  R     : 블록 보관       │
└──────────────────┴──────────────────────────┘
```

### 물리 시스템

- **중력 + 충돌**: 블록이 물리 엔진으로 낙하, 바닥/기존 블록과 충돌 감지
- **360도 자유 회전**: Q/E키로 angularVelocity를 가하면 마찰에 의해 자연스럽게 감속
- **90도 즉시 회전**: ↑키로 전통 테트리스 스타일 회전 (충돌 시 무시)
- **블록별 고유 색상**: I(#00f0f0), J(#0000f0), L(#f0a000), O(#f0f000), S(#00f000), Z(#f00000), T(#a000f0)
- **라인 절단**: 기울어진 블록이 라인에 걸쳐있으면 수평으로 절단됨
- **절단 후 재낙하**: 절단된 위 조각이 다시 물리 적용받아 떨어짐

## Tab 2 — 🧩 Why Tetris (주제 선정 이유)

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  왜 물리 테트리스인가?                        │
├─────────────────┬───────────────────────────┤
│                 │                           │
│  [일반 테트리스] │  [물리 테트리스]          │
│  블록 = 그리드  │  블록 = Component         │
│  단위 이동      │  State = 위치/각도/속도    │
│                 │  useEffect = 게임 루프     │
│                 │  useMemo = 충돌 캐싱       │
│                 │                           │
├─────────────────┴───────────────────────────┤
│  컴포넌트 트리 시각화                         │
│                                             │
│  <TetrisApp>                                │
│  ├── <Board />    ← 보드 상태만 받음 (props) │
│  ├── <Block />    ← 위치/각도만 받음 (props) │
│  ├── <Score />    ← 점수만 받음 (props)      │
│  └── <Preview />  ← 다음 블록만 받음 (props) │
│                                             │
│  → State는 TetrisApp(루트)에서만 관리        │
│    (Lifting State Up 패턴)                  │
├─────────────────────────────────────────────┤
│  라인 클리어 시연                             │
│                                             │
│  [ 시연 실행 ] → 기울어진 블록이 라인에 걸침  │
│               → cutPieceAtLine() 호출       │
│               → 수평 절단 애니메이션         │
│               → 위 조각만 남아 재낙하        │
└─────────────────────────────────────────────┘
```

### 핵심 포인트

- **"블록 하나 = Component 하나"** 를 트리로 시각화
- **Lifting State Up** 패턴을 코드 + 다이어그램으로 설명
- 라인 절단 로직을 **슬로우 모션 시연**으로 보여줌

## Tab 4 — ⚙️ 학습 (우리가 발견한 것들)

### 핵심 방향

> "구현하다 보니 이런 문제가 생겼고, 그걸 해결하기 위해 이 기능을 추가했습니다"

---

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  Hook을 게임에 어떻게 배치했나?               │
├─────────────────────────────────────────────┤
│                                             │
│  useState                                   │
│  ├── 게임 전체 상태 (PhysicsState)          │
│  ├── 다음 블록 (nextPiece)                  │
│  └── 보관 블록 (heldPiece, 1개 제한)        │
│                                             │
│  useEffect                                  │
│  ├── 게임 루프 시작 (mount 시)              │
│  ├── 블록 착지 감지                         │
│  ├── 키보드 이벤트 등록                     │
│  │   (←→ 이동, ↑ 90도회전, Q/E 자유회전,  │
│  │    ↓ 소프트드롭, SPC 하드드롭, R 홀드)  │
│  └── cleanup → 루프/이벤트 해제             │
│                                             │
│  useMemo                                    │
│  └── 충돌 감지 결과 캐싱                    │
│      블록 위치 바뀔 때만 재계산             │
│                                             │
│  Batching                                   │
│  └── 이동(x) + 중력(y) + 회전(angle)       │
│      setState 3번 → 렌더링 1번             │
│                                             │
├─────────────────────────────────────────────┤
│  개발하면서 발견한 문제들                     │
├─────────────────────────────────────────────┤
│                                             │
│  🔴 문제 1. 블록이 움직일 때마다 버벅임       │
│     원인: setState 3번 → 렌더링 3번         │
│     해결: Batching 구현                     │
│     → 같은 tick 안의 setState 묶어서 1번   │
│                                             │
│  🔴 문제 2. 게임 재시작 시 루프 중복 실행    │
│     원인: useEffect cleanup 미구현          │
│     해결: cleanup에서 루프 cancelAnimation  │
│     → mount/unmount 생명주기 완성           │
│                                             │
│  🔴 문제 3. 60fps에서 충돌 계산 과부하      │
│     원인: 매 프레임 전체 블록 충돌 재계산   │
│     해결: useMemo로 deps 변경 시만 재계산  │
│                                             │
│  🔴 문제 4. hooks[] 순서 꼬임              │
│     원인: hookIndex 초기화 누락             │
│     해결: mount/update 시 hookIndex = 0    │
│     → "함수가 매번 실행돼도 상태 유지"의 핵심│
│                                             │
│  🔴 문제 5. R키 연속 입력 시 블록 무한 교체  │
│     원인: canHold 플래그 없음               │
│     해결: 착지 시 canHold = true 복원      │
│     → 실제 테트리스 룰과 동일한 제약        │
│                                             │
├─────────────────────────────────────────────┤
│  실제 React와 비교해서 배운 것               │
├─────────────────────────────────────────────┤
│                                             │
│  우리가 구현한 것        실제 React 대응     │
│  ─────────────────────────────────────────  │
│  FunctionComponent      함수형 컴포넌트      │
│  hooks[] + hookIndex    Fiber.memoizedState │
│  mount() / update()     Reconciler          │
│  Batching               automatic batching  │
│  useEffect cleanup      componentWillUnmount│
│                                             │
│  → "우리가 겪은 문제가 React가                │
│     이 기능을 만든 이유와 동일했습니다"       │
│                                             │
└─────────────────────────────────────────────┘
```

### 핵심 포인트

- **문제 → 해결 서사** — "이렇게 만들었다"가 아니라 "이런 문제를 발견해서 이걸 추가했다"
- **게임 기능과 Hook 1:1 매핑** — 다음 블록, 보관 블록, 착지, 이동이 각각 어느 Hook에 연결됐는지
- **React 대응표는 마지막에** — 문제를 직접 겪은 후 "그게 React가 만들어진 이유였다" 로 마무리

3. Tab "flamegraph" → 실시간 Flamegraph 메트릭 패널

전체 레이아웃:
- 상단 네비게이션 탭
- 탭 전환은 해시 기반 라우팅 (#play, #why, #flamegraph, #lifecycle)
- Tab 1에서 게임 플레이 중 Tab 3 Flamegraph에 실시간 반영

1. FunctionComponent로 루트 TetrisApp 구성

   - Space : 하드 드롭 (hardDrop)
   - R : 블록 보관 (holdPiece)
5. 자식 컴포넌트는 props만 받는 순수 함수
6. FunctionComponent 렌더링 시간을 Flamegraph record()로 전달

완료 후 커밋하고 feat/app에 푸시해줘.
그 다음 feat/app → dev로 PR 생성해줘.

feat(app): 4탭 SPA 물리 테트리스 통합
